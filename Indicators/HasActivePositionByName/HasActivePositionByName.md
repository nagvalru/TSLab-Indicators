# Есть активная позиция по имени

## Назначение

Пользовательский C# Handler для TSLab, который проверяет наличие активной позиции, открытой конкретным блоком открытия позиции.

## Первое использование

Индикатор создан для проекта `ai-agent`. Первая стратегия использования: простая SMA-стратегия в папке `AI` с выводом блока на график.

## Расчет

На каждом баре блок берет список позиций из входного `SECURITY` и вызывает `GetLastActiveForSignal(EntryBlockName, bar)`. Если найденная позиция активна на этом баре, блок возвращает `true`; иначе возвращает `false`.

## Входы

`Source` - источник инструмента `SECURITY`, такой же вход, как у штатного блока `Есть активная позиция`.

## Параметры

`EntryBlockName` - строка с точным именем блока открытия позиции. Например, если блок открытия называется `openLong`, параметр должен быть равен `openLong`.

## Выход

Логический ряд `BOOL`.

## Применение

Если в скрипте есть несколько блоков открытия позиции, например `openLong`, `openShort`, `openLong1`, `openShort2`, этот handler позволяет отдельно проверить активность позиции только для одного выбранного блока.

## Особенности

Сравнение имени выполняется по точной строке после удаления пробелов в начале и конце параметра. Блок работает с позициями текущего скрипта и текущего источника `SECURITY`.

В DLL экспортируются два handler-типа:

- `HasActivePositionByName` - совместимость с уже вставленными блоками.
- `ЕстьАктивнаяПозицияПоИмени` - русское имя для поиска в панели инструментов по слову `Есть`.

Реализация использует `IBar2BoolHandler`, как штатный блок `Есть активная позиция`, и рассчитывает значение только для текущего бара. Это быстрее, чем полный пересчет `IList<bool>` внутри handler.
