# KaufmanAdaptiveHighest

## 1. Название индикатора

- Имя: `KaufmanAdaptiveHighest`
- Короткое пояснение: адаптивный максимум на переменном периоде

## 2. Назначение

Индикатор считает максимум ряда `source` на окне переменной длины. Длина окна на каждом баре берётся из входного ряда `varPeriod`.

Он нужен там, где обычного `Highest` с фиксированным периодом уже недостаточно, а окно экстремума должно меняться вместе с текущим адаптивным периодом стратегии.

## 3. Проект и стратегия первого использования

- Проект первого использования: `KaufmanStrategies`
- Стратегия первого использования: `TSM Dynamic Breakout-Volatility System`

## 4. Тип индикатора

Это расчётный адаптивный индикатор. Он работает не с фиксированным числом баров, а с периодом-серией.

## 5. Логика расчёта

На каждом баре индикатор:

1. берёт текущее значение `varPeriod`
2. округляет его до целого значения по обычному математическому правилу
3. ограничивает период минимумом `1`
4. просматривает последние `period` баров исходного ряда `source`
5. возвращает максимум на этом окне

Формально:

`result[i] = max(source[j])`, где `j` пробегает окно от `i - period + 1` до `i`

## 6. Входы индикатора

- `source`
  - тип: числовой ряд
  - смысл: ряд, по которому ищется максимум
- `varPeriod`
  - тип: числовой ряд
  - смысл: адаптивная длина окна
  - ограничение: фактическая длина окна не может быть меньше `1`

## 7. Выходы индикатора

Индикатор возвращает один числовой ряд: максимум исходного ряда на адаптивном окне.

## 8. Параметры пользователя

Отдельных скалярных параметров у самого индикатора нет. Управление идёт через входной поток `varPeriod`.

Оптимизация в стратегии выполняется не внутри этого блока, а через параметры, которые формируют `varPeriod`.

## 9. Применение в стратегиях

В стратегии `TSM Dynamic Breakout-Volatility System` этот индикатор используется для построения адаптивных верхних рубежей входа и выхода:

- `highestAdjEntry`
- `highestHigh`
- `highestAdjExit`
- `highestHighB`

Типичный сценарий применения: стратегия уже рассчитала адаптивный период и теперь хочет получить верхний экстремум именно на этом изменяющемся окне.

## 10. Ограничения и особенности реализации

- Период округляется до целого.
- Минимальный период принудительно равен `1`.
- Если на ранних барах истории полного окна ещё нет, используется доступная часть истории.
- Корректность сильно зависит от того, чтобы upstream-формула давала разумные значения периода.

## 11. Проверка корректности

Быстрая проверка:

- подать константный `varPeriod`, например `10`
- сравнить результат с обычным `Highest(source, 10)`

Если оба ряда совпадают, базовая логика работает правильно.

## 12. Неоднозначности

Индикатор сам по себе неоднозначностей не содержит. Все смысловые вопросы обычно относятся не к нему, а к тому, как именно стратегия формирует ряд `varPeriod`.
